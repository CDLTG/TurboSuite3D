(General Revit 2025 C# Add-In)
________________________________________
1. Revit & API Context
•	Revit Version: 2025
•	Language: C#
•	API Type: ExternalCommand
•	Transaction model: Single transaction
Constraints:
•	No manifest files
________________________________________
2. Preconditions
•	Required active view Floor Plan or RCP
•	Required element selection Lighting Fixture
•	Minimum 1 Lighting Fixture selection, no maximum
Define behavior if preconditions are not met:
•	Cancel command
•	Show user message
________________________________________
3. User Workflow
1.	User selects 1 or more Lighting Fixtures
2.	User launches the command
3.	User indicates a direction for tag placement by pressing one of the arrow keys
4.	Add-in places tag at each selected Lighting Fixture in the indicated direction
5.	Command completes with feedback
________________________________________
4. Plug-in Logic
•	Evaluate each selected Lighting Fixture for rotation/orientation. All tags generated by the plug-in should visually match the rotation of it's host Lighting Fixture. X and Y axis directions should be considered relative to the Lighting Fixture and not relative to the project X and Y axis.
•	Place Lighting Fixture Tag 'AL_Tag_Lighting Fixture (Type)' at each selected Lighting Fixture. If 'AL_Tag_Lighting Fixture (Type)' does not exist in the project, cancel the plug-in.
•	Evaluate each of the generated Lighting Fixture Tags. If the tag is 2 or less characters long, assign a BoundingBox value of 7.75". If the tag is 3 characters long, assign a BoundingBox value of 11.15". If the tag is 4 or more characters long, assign a BoundingBox value equal to the bounding box of the tag.
•	Access each selected Lighting Fixture and extract the parameter values for "Symbol Length" and "Symbol Width". If these parameters do not exist, use a default value of 4.5" for each.
•	Offset the generated tags in the direction indicated by the user. 
•	For 'up' and 'down' directions, offsets will be for Y-axis values. Offset distance is ((Symbol Length / 2) + 7.75") and ((Symbol Length / 2) - 7.75") respectively
•	For 'right' and 'left' directions, offsets will be for X-axis values. Offset distance is ((Symbol Width / 2) + (BoundingBox / 2) + 2.32") and ((Symbol Width / 2) - (BoundingBox / 2) - 2.32") respectively
