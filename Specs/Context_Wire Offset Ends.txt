Moving Wire Offset Ends in Revit API
Overview
When working with Revit electrical wires, each wire has special grip points called "Offset Ends" at the start and end of the wire. These are distinct from regular vertices - when you hover over them in Revit, the tooltip shows "Wire: Change Offset End" rather than "Wire: Move Vertex". Offset ends allow the wire to visually extend beyond its connected point while maintaining electrical connectivity to the fixture.
The Problem
The Revit API does not expose a direct method like Wire.SetOffsetEnd() to manipulate these offset ends. Attempting to use parameters like "Start Offset X" or "End Offset X" via LookupParameter() also does not work.
The Solution
The offset ends can be manipulated by calling Wire.SetVertex() twice in sequence on the same vertex index. The key insight is that:

The first SetVertex() call moves the vertex to a position
The second SetVertex() call on the same index moves the offset end relative to that vertex

Important Constraint
You cannot set a vertex to a position that already has a vertex (within tolerance). This means the first SetVertex() call must move the vertex to a position that doesn't overlap with an existing vertex.
Implementation Pattern
csharp// Get the wire's vertex count
int vertexCount = wire.NumberOfVertices;

// Define offsets
double initialOffset = 0.5 / 12.0;   // 0.5" - intermediate position to avoid overlap
double finalOffset = 2.5 / 12.0;     // 2.5" - final offset end position

// Move the START offset end (vertex index 0)
// Step 1: Move vertex to intermediate position (away from any existing vertex)
wire.SetVertex(0, connectorOrigin + offsetDirection * initialOffset);
// Step 2: Move again to set the offset end at final position
wire.SetVertex(0, connectorOrigin + offsetDirection * finalOffset);

// Move the END offset end (last vertex index)
// Step 1: Move vertex to intermediate position
wire.SetVertex(vertexCount - 1, connectorOrigin + offsetDirection * initialOffset);
// Step 2: Move again to set the offset end at final position
wire.SetVertex(vertexCount - 1, connectorOrigin + offsetDirection * finalOffset);
Complete Example
csharpusing (Transaction t = new Transaction(doc, "Create Wire"))
{
    t.Start();

    // Create the wire
    Wire wire = Wire.Create(doc, wireType.Id, view.Id, WiringType.Arc, points, connector1, connector2);

    // Define the direction to offset (e.g., wall normal for wall-hosted fixtures)
    XYZ offsetDirection = wallNormal;
    
    double initialOffset = 0.5 / 12.0;   // 0.5"
    double finalOffset = 2.5 / 12.0;     // 2.5"
    int vertexCount = wire.NumberOfVertices;

    // Set start offset end
    wire.SetVertex(0, connector1.Origin + offsetDirection * initialOffset);
    wire.SetVertex(0, connector1.Origin + offsetDirection * finalOffset);

    // Set end offset end
    wire.SetVertex(vertexCount - 1, connector2.Origin + offsetDirection * initialOffset);
    wire.SetVertex(vertexCount - 1, connector2.Origin + offsetDirection * finalOffset);

    t.Commit();
}
```

## Key Points

1. **Two calls required**: You must call `SetVertex()` twice on the same index to move the offset end
2. **Avoid position overlap**: The first call must move to a position that doesn't coincide with an existing vertex (use a small intermediate offset)
3. **Vertex indices**: Use index `0` for the start offset end and `wire.NumberOfVertices - 1` for the end offset end
4. **Electrical connectivity preserved**: The wire remains electrically connected to the fixtures even though the visual endpoint is offset
5. **Direction vector**: The offset direction is typically perpendicular to the wall or host surface (e.g., wall normal for wall-mounted fixtures)

## Common Error

If you see this error:
```
"The vertex point cannot be added to the wire because there is already a vertex at this position on the view plane (within tolerance)."
This means the first SetVertex() call is trying to place the vertex at a position that overlaps with an existing vertex. Increase the initialOffset value or ensure you're moving in a direction that doesn't coincide with other vertices.
